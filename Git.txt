	1.	Создание нового репозитория на GitHub:
			Зайдите на  и авторизуйтесь.
			Нажмите «+» в верхнем правом углу → «New repository».
			Repository name: TCP-UDP-Server (как у вас).
			Description: (по желанию).
			Public (если проект открытый).
			Снимите галочку с «Initialize this repository with:» (не нужно README/LICENSE).
			Нажмите «Create repository».
		
	2.	Подготовка локальной папки:
			mkdir -p /a/NJ/2025/НДМ Системы/Server
			cd /a/NJ/2 Newton/2025/НДМ Системы/Server
			Перенос/копирование в неё нужных файлов.	cp /путь/к/сохранённым/файлам/* .
			
	3.	Инициализация Git и связь с GitHub:
			git init
			git add .
			git commit -m "Initial commit: базовые файлы проекта"
			git remote add origin https://github.com/VV0land/TCP-UDP-Server.git
			git push --force -u origin master			-u — связывает ветку master локально и удалённо (нужно один раз),
														--force — стирает все предыдущие коммиты на сервере и заменяет их вашими.

			git add .
			git status
			?	git reset HEAD x64/Release/				Убрать из индекса ненужные файлы
				
			git rm -r --cached x64/						Убериать папки из индекса (но оставьте на диске)	-r — рекурсивно -cached — удалить только из индекса Git, не трогая реальные файлы на диске.
			и/или добавить x64/ в	.gitignore 
			
			git status
			git commit -m "переделка того-то"			!!! Если коммит не идет(особенно в ПЕРВЫЙ РАЗ), то	git config --global user.email "you@example.com" и git config --global user.name "Your Name"
			
			git log --oneline							Должен появиться новый коммит с сообщением.
